# 反射和内置方法

## 一、什么是反射（reflection）？

>反射的概念是由 Smith在 1982 年首次提出的，主要指的是程序可以访问、检测以及修改它本身状态或行为的一种能力（自省）。
>
>这一概念的提出很快引发了计算机科学领域关于应用反射性的研究，它首先被程序语言的设计领域所采用，并在 Lisp 和面向对象方面取得了成绩。

## 二、Python 中的反射

>Python是动态语言，而 反射（reflection） 机制被视为 动态语言 的关键。
>
>在 Python中，反射主要指的是：通过字符串的形式操作对象相关的属性。Python 中一切皆对象，因此都可以使用反射。
>
>更加详细的说，反射机制指的是在程序的运行状态中：对于任意的对象，都可以获取或设置其属性和方法，这种 动态获取程序信息以及动态调用对象的功能 ，我们称之为 反射（reflection）。
>
>使用反射的过程中，主要涉及到了 4个可以实现自省的内置函数。

## 三、关于反射的内置函数

>1、*hasattr(__obj, __name)*
>
>2、*getattr(__o, name, __default)*
>
>3、*setattr(__obj, __name, __value)*
>
>4、*delattr(__obj, __name)*
>
>以上4个内置函数，是Python 提供给我们的用于实现反射的内置函数

### 1、`hasattr(__obj, __name)`

>简介：该内置函数用于返回对象是否具有给定名称的属性，其实质是通过调用 getattr(__o, name)并捕获 *AttributeError* 来完成的。

```python
class Person(object):
    """ 人类 """

    country = 'China'


print(hasattr(Person, 'name'))
print(hasattr(Person, 'country'))

# 结果
False
True
```

### 2、`getattr(__o, name, __default)`

>简介：该内置函数用于从对象获取给定名称的属性，*getattr(x, 'y')* 等效于 *x.y* 。
>
>当给定 *default* 参数的时候，如果属性不存在，则返回 *default* 参数
>
>如果没有给定 *default*参数，如果属性不存在，那么将抛出异常！
>
>请看如下实例：

#### 1.没有给定default参数

```python
class Person(object):
    """ 人类 """

    country = 'China'


print(getattr(Person, 'name'))


# 结果
Traceback (most recent call last):
  File "E:\工作\xiaowu\草稿\test.py", line 7, in <module>
    print(getattr(Person, 'name'))
          ~~~~~~~^^^^^^^^^^^^^^^^
AttributeError: type object 'Person' has no attribute 'name'
```

#### 2.给定了default参数

```python
class Person(object):
    """ 人类 """

    country = 'China'


print(getattr(Person, 'name', '非常抱歉！该属性不存在哦！'))


# 结果
非常抱歉！该属性不存在哦！
```

### 3、`setattr(__obj, __name, __value)`

>简介：给内置函数用于给对象设置给定名称的属性，*setattr(x, 'y', z)* 等价于 *x.y = z* 。

```python
class Person(object):
    """ 人类 """

    country = 'China'


print(hasattr(Person, '__tag'))
setattr(Person, '__tag', '张三')
print(hasattr(Person, '__tag'))
print(getattr(Person, '__tag', '非常抱歉！该属性不存在哦！'))


# 结果
False
True
张三
```

4、`delattr(__obj, __name)`

>简介：该内置函数用于从对象中删除给定的属性，*delattr(x, 'y')* 等价于 *del x.y* 。

```python
class Person(object):
    """ 人类 """

    country = 'China'


print(hasattr(Person, 'country'))
delattr(Person, 'country')
print(hasattr(Person, 'country'))


# 结果
True
False
```

## 四、反射的应用

### 1、反射当前模块成员

```python
import sys


def f1():
    """
    f1 函数
    :return:
    """
    print(f'--- 我是函数 f1 ---')


def f2():
    """
    f2 函数
    :return:
    """
    print(f'--- 我是函数 f2 ---')


this_module = sys.modules[__name__]
print(this_module)
print(hasattr(this_module, 'f1'))
print(getattr(this_module, 'f2', '非常抱歉！该属性不存在哦！'))
```

## 五、为什么要使用反射

>由于反射机制的存在，使得我们可以更加灵活的操作程序。具体来讲存在以下两个好处：
>
> 实现可插拔机制
>
> 动态导入模块

### 1、实现可插拔机制

>场景：有两个程序员 ，一个叫张三，另外一个叫李四。张三在写程序的时候需要使用李四所写的类，但是李四由于有事情，不能及时的完成自己的类，因此张三想到了反射，使用反射机制，张三可以继续完成自己的代码，而无需等待李四回来以后再写程序。
>
>李四有事情无法完成自己所写的类

>模块：ftp_client_lisi.py

```python
class FTPClient(object):
    """ FTP 客户端，但是还没有实现具体的功能 """

    def __init__(self, addr):
        print(f'正在连接服务器：{addr}')
        self.addr = addr
```

>张三需要李四所写的类

>模块：main.py

```python
import ftp_client_lisi

ftp_client = ftp_client_lisi.FTPClient('10.211.55.3')
if not hasattr(ftp_client, 'get'):
    print(f'不存在此方法')
    print(f'处理其它逻辑。。。')
else:
    func_get = getattr(ftp_client, 'get', None)
    func_get()

```

## 六、内置方法

>Python的Class机制内置了很多特殊的方法来帮助使用者高度定制自己的类，这些内置方法都是以双下划线开头和结尾的，会在满足某种条件时自动触发，我们以常用的 *__str__* 和 *__del__* 为例来简单介绍它们的使用。

### 1、` __str__() & __repr__()`

>`__str__`方法会在对象被打印时自动触发，print功能打印的就是它的返回值，我们通常基于方法来定制对象的打印信息，该方法必须返回字符串类型。
>
>`__repr__()` 方法用于在交互式界面直接输入对象，会触发 `__repr__()` 的执行并将其返回值作为打印结果。

```python
class People:
    def __init__(self,name,age):
        self.name = name
        self.age = age
    def __str__(self):
        return '<Name:%s Age:%s>' %(self.name, self.age)  # 返回类型必须是字符串

p = People('lili', 18)
print(p) # 触发p.__str__()，拿到返回值后进行打印


# 结果
<Name:lili Age:18>
```

>总结：
>
>如果同时存在，print使用str，终端使用repr
>
>如果只有str，print使用str，终端将打印内存地址
>
>如果只有repr，print使用repr，终端使用repr

### 2、` __del__()`

>`__del__` 会在对象被删除时自动触发。由于 Python 自带的垃圾回收机制会自动清理 Python 程序的资源，所以当一个对象只占用应用程序级资源时，完全没必要为对象定制__del__方法，但在产生一个对象的同时涉及到申请系统资源（比如系统打开的文件、网络连接等）的情况下，关于系统资源的回收，Python的垃圾回收机制便派不上用场了，需要我们为对象定制该方法，用来在对象被删除时自动触发回收系统资源的操作。

```python
class MySQL:
    def __init__(self,ip,port):
        self.conn=connect(ip,port) # 伪代码，发起网络连接，需要占用系统资源
    def __del__(self):
        self.conn.close() # 关闭网络连接，回收系统资源

obj=MySQL('127.0.0.1',3306) # 在对象obj被删除时，自动触发obj.__del__()
```
