# 仓库设置介绍

## 一、属性设置

### 1、Repository Options（存储库选项）

#### 1.**状态（State）**

> **Active**：用于正常开发和维护中的项目或分支，允许提交和合并。
>
> **Read Only**：适用于不再活跃但仍需保留查看权限的项目或分支（如归档版本、历史版本）。
>
> **Hidden**：用于临时不对外公开或尚未发布的项目或分支，适合需要隐藏的敏感内容或正在开发的功能。

#### 2.**提交类型（Submit type）**：

>**Inherit (Merge if necessary)**：继承上一级配置，默认合并或快进。
>
>**Merge if necessary**：需要时合并，不需要时快进。
>
>**Fast forward only**：仅快进，无法快进则拒绝合并。
>
>**Rebase Always**：总是执行 rebase。
>
>**Rebase if necessary**：必要时执行 rebase。
>
>**Merge always**：始终执行合并操作，无论能否快进。
>
>**Cherry pick**：只应用特定提交，不进行完整合并。

#### 3.**允许内容合并（Allow content merges）**：

> 是否允许在解决冲突时自动进行内容合并。

#### 4.**为目标分支中不存在的每个提交创建新变更（Create a new change for every commit not in the target branch）**：

> 是否为每个不在目标分支中的提交创建新的变更，而不是将所有提交合并在一起。

#### 5.**要求提交消息中包含 Change-Id（Require Change-Id in commit message）**：

> 强制要求提交消息中包含唯一的 Change-Id，以便跟踪变更。

#### 6.**当变更推送进行审查时，拒绝隐式合并（Reject implicit merges when changes are pushed for review）**：

> 阻止在没有明确审查和批准的情况下自动合并变更。

#### 7.**允许将未注册用户添加为评审人和抄送人（Enable adding unregistered users as reviewers and CCs on changes）**：

> 允许将未注册的用户添加为评审人或抄送接收变更通知。

#### 8.**默认将所有新变更设置为私有（Set all new changes private by default）**：

> 默认将新提交的变更标记为私有，仅特定用户可查看。

#### 9.**默认将新变更标记为“正在进行中”（Set new changes to "work in progress" by default）**：

> 将新提交的变更默认标记为“正在进行中”，即变更尚未准备好进行最终审查。

#### 10.**Git 对象大小限制（Maximum Git object size limit）**：

> 限制可以推送到仓库的 Git 对象（如文件或提交）的最大大小。

#### 11.**提交时匹配作者日期和提交者日期（Match authored date with committer date upon submit）**：

> 提交变更时，将作者日期调整为与提交者日期一致。

#### 12.**提交时拒绝空提交（Reject empty commit upon submit）**：

> 防止提交没有任何更改（空提交）的代码。

### 2、Contributor Agreements（贡献者协议）

#### 1.**要求有效的贡献者协议才能上传（Require a valid contributor agreement to upload）**：

> 确保贡献者在上传变更前必须拥有有效的协议。

#### **2.提交消息中需要“Signed-off-by”行（Require Signed-off-by in commit message）**：

> 强制要求提交消息中包含“Signed-off-by”行，表明作者同意项目的贡献政策条款。



## 二、权限设置

### 1、Global Capabilities（全局权限）

>**Administrate Server**（管理服务器）：允许用户执行全局管理任务，如创建项目、管理用户、配置系统等。拥有这个权限的用户是 Gerrit 实例的超级管理员。
>
>**Priority**（优先级）：赋予用户对 Gerrit 实例的任务或操作设定优先级的能力，比如构建或提交的处理顺序。
>
>**Stream Events**（事件流）：允许用户订阅 Gerrit 的事件流，以实时跟踪项目中的变更和活动。常用于集成外部工具或自动化系统。

### 2、Reference: refs/*

>这类权限应用于 Git 仓库中的所有引用（ref），包括分支和标签。
>
>- **Read**（读取）：允许用户查看所有引用下的内容，如查看代码、分支、提交历史等。

### 3、Reference: refs/for/*

>这个引用通常用于 Gerrit 中的代码审查流程，指向推送用于审查的变更。
>
>- **Add Patch Set**（添加补丁集）：允许用户向 Gerrit 提交补丁集进行代码审查。这通常意味着用户可以推送变更到 `refs/for/*`，从而触发审查流程。

### 5、Reference: refs/for/refs/*

>**Push**（推送）：允许用户将变更推送到 Gerrit，用于代码审查的 refs（引用）。
>
>**Push Merge Commit**（推送合并提交）：允许用户推送包含合并提交的变更。合并提交通常在处理多个分支时产生，拥有这个权限的用户可以推送这些特殊类型的提交。

### 6、Reference: refs/heads/*

>这是对分支（如 `refs/heads/master` 或其他具体分支）进行操作的权限。
>
>- **Create Reference**（创建引用）：允许用户在 `refs/heads/*` 下创建新分支。
>- **Edit Topic Name**（编辑主题名称）：允许用户修改变更的主题名称，通常用于对多个变更进行分组或标记。
>- **Forge Author Identity**（伪造作者身份）：允许用户在提交时使用不同于实际提交者的作者信息，通常用于代表其他人提交代码。
>- **Forge Committer Identity**（伪造提交者身份）：允许用户伪造提交者身份，以便在特定情况下更改提交者的身份信息。
>- **Label Code-Review**（标签代码审查）：允许用户为变更添加代码审查分数（如 `Code-Review +2` 或 `Code-Review -2`），表示对代码的审查结果。
>- **Push**（推送）：允许用户推送代码到具体的分支中，如 `refs/heads/master`，适用于直接推送无需经过代码审查的场景。
>- **Read**（读取）：允许用户查看该分支下的所有内容，包括代码、提交历史等。
>- **Revert**（回滚）：允许用户对已提交的代码进行回滚操作，撤销指定的提交。
>- **Submit**（提交）：允许用户将通过代码审查的变更合并到目标分支中。

### 7、Reference: refs/meta/config

>这个引用是 Gerrit 项目的配置文件存储位置。
>
>- **Create Reference**（创建引用）：允许用户创建或修改该配置文件的引用。
>- **Label Code-Review**（标签代码审查）：允许用户为项目配置的变更添加代码审查分数。
>- **Push**（推送）：允许用户推送对项目配置的变更。
>- **Read**（读取）：允许用户查看项目的配置文件。
>- **Submit**（提交）：允许用户将通过审查的项目配置变更合并到仓库中。

### 8、Reference: refs/meta/version

>**Read**（读取）：允许用户查看 Gerrit 版本控制相关的引用。这通常用于查看项目的元数据和版本信息。

### 9、Reference: refs/tags/*

>这是与标签（tags）相关的权限，用于创建、签名和注释标签。
>
>- **Create Reference**（创建引用）：允许用户创建新的 Git 标签。
>- **Create Signed Tag**（创建签名标签）：允许用户创建带有 GPG 签名的标签，常用于安全发布和验证。
>- **Create Annotated Tag**（创建注释标签）：允许用户创建带有注释的标签，附加更多的元信息（如描述、作者、时间等）。
