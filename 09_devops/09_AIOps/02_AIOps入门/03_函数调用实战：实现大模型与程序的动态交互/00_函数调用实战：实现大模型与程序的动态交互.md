# 函数调用实战：实现大模型与程序的动态交互

![思维导图](./%E6%80%9D%E7%BB%B4%E5%AF%BC%E5%9B%BE.png)

## 一、什么是 Function Calling？

**Function Calling（函数调用）** 是大模型（如 GPT-4）与外部程序交互的一种机制。它允许大模型根据用户的自然语言输入，**智能地选择并建议调用某个预定义的函数**，并自动填充该函数所需的参数。

> ⚠️ 注意：大模型本身不能直接执行代码！它只是“建议”要调用哪个函数，并提供参数。真正的函数执行需要由你的后端代码来完成。

### 1、图解：Function Calling 工作流程

```
[用户输入] 
     ↓
[大模型分析语义]
     ↓
[决定调用哪个函数？例如：get_log()]
     ↓
[生成函数名 + 参数 JSON]
     ↓
[你的代码接收结果 → 真正执行函数]
     ↓
[获取函数返回值 → 再次交给大模型推理]
     ↓
[最终回答用户]
```

这个过程实现了 **“AI 理解需求” + “程序执行任务”** 的完美结合。

## 二、为什么需要 Function Calling？——对比 JSON Mode 的局限性

我们先来看一个常见问题：如果不用 Function Calling，该怎么让 AI 返回结构化数据？

### ❌ 方法1：使用 JSON Mode（固定格式输出）

假设你想让 AI 根据用户请求返回特定格式的 JSON 数据：

```json
{
  "action": "restart_service",
  "service_name": "payment"
}
```

你需要在 `system prompt` 中明确告诉 AI：“当用户说重启服务时，返回这样的 JSON”。

但问题来了：

| 问题                 | 说明                                                         |
| -------------------- | ------------------------------------------------------------ |
| ✖️ 场景太多无法枚举   | 用户可能说：“重启支付服务”、“把 payment 搞一下”、“服务挂了快重开”……你怎么写完所有情况？ |
| ✖️ 参数动态变化难处理 | 时间范围、服务名称等都是变量，难以用静态 schema 描述。       |
| ✖️ 扩展性差           | 每新增一个功能，都要改提示词（prompt），维护成本高。         |

📌 **结论：JSON Mode 只适合简单、固定的结构化输出，不适合复杂业务逻辑。**

## 三、Function Calling 如何解决这些问题？

Function Calling 的核心思想是：

> **把“做什么”和“怎么做”分离**  
>
> - 大模型负责判断“**该调哪个函数**”（做什么）  
> - 开发者编写函数实现“**具体怎么执行**”（怎么做）

### Function Calling 的优势

| 优势           | 说明                                                    |
| -------------- | ------------------------------------------------------- |
| ✔️ 动态识别意图 | 不管用户怎么说，AI 都能理解你要查日志、重启服务等。     |
| ✔️ 自动提取参数 | 能从句子中抽取出 `service=payment`, `time=2天` 等信息。 |
| ✔️ 易于扩展     | 新增功能只需添加新函数，无需修改系统提示语。            |
| ✔️ 安全可控     | 函数由你控制，不会随意执行危险操作。                    |

## 四、实战演示：快速上手 Function Calling

下面我们通过一个简单的例子来演示整个流程。

### 1、示例目标：

用户输入：“查看 app=graph 且包含 error 的日志”  
→ AI 应识别出要调用 `analyze_lucky_log(query_str)` 函数  
→ 并传入参数：`query_str="app=graph && keyword=error"`

### 第一步：定义工具函数（Tools）

我们在代码中定义一个函数及其描述，告诉 AI 它能做什么：

```python
tools = [
    {
        "type": "function",
        "function": {
            "name": "analyze_lucky_log",
            "description": "从日志系统中查询符合条件的日志条目",
            "parameters": {
                "type": "object",
                "properties": {
                    "query_str": {
                        "type": "string",
                        "description": "日志查询字符串，例如 'app=graph && keyword=error'"
                    }
                },
                "required": ["query_str"]
            }
        }
    }
]
```

📌 **关键字段解释：**

| 字段          | 含义                                   |
| ------------- | -------------------------------------- |
| `name`        | 函数名，用于后续匹配                   |
| `description` | 告诉 AI 这个函数是干啥的（非常重要！） |
| `parameters`  | 定义参数结构，AI 会据此提取内容        |
| `required`    | 必填参数列表                           |

---

### 第二步：构造对话消息（Messages）

```python
messages = [
    {"role": "system", "content": "你是 Lucky 日志分析助手，可以调用函数帮助用户分析日志。"},
    {"role": "user", "content": "查看 app=graph 且关键字包含 error 的日志"}
]
```

---

### 第三步：调用大模型接口（如 OpenAI API）

```python
response = client.chat.completions.create(
    model="gpt-4o",
    messages=messages,
    tools=tools,
    tool_choice="auto"  # 让模型自动选择是否调用函数
)
```

---

### 第四步：解析模型返回结果

模型不会直接返回答案，而是返回一个“函数调用建议”：

```json
{
  "tool_calls": [
    {
      "function": {
        "name": "analyze_lucky_log",
        "arguments": "{\"query_str\": \"app=graph && keyword=error\"}"
      }
    }
  ]
}
```

📌 注意：`arguments` 是一个 JSON 字符串，需解析后得到实际参数。

### 第五步：在代码中真正执行函数

```python
import json

# 解析参数
args = json.loads(response.tool_calls[0].function.arguments)
query = args["query_str"]

# 真正执行函数
def analyze_lucky_log(query_str):
    print(f"函数调用的参数是：{query_str}")
    return "示例日志：[ERROR] graph service timeout"

result = analyze_lucky_log(query)
```

### 第六步：将函数结果再次传给大模型做最终回复

```python
messages.append({
    "role": "assistant",
    "content": None,
    "tool_calls": [response.tool_calls[0]]
})

# 添加函数执行结果
messages.append({
    "role": "tool",
    "content": result,
    "tool_call_id": response.tool_calls[0].id
})

# 再次请求模型生成最终回答
final_response = client.chat.completions.create(
    model="gpt-4o",
    messages=messages
)

print(final_response.choices[0].message.content)
```

最终输出可能是：

> “我已经查到关键字包含 error 且应用为 graph 的日志：  
> [ERROR] graph service timeout  
> 是否需要进一步过滤或查看详情？”

## 六、典型应用场景

| 场景       | 说明                                        |

## 七、总结与最佳实践

### 1、使用 Function Calling 的最佳方式：

1. **清晰定义每一个函数的功能和参数**
2. **给函数写高质量的 description（决定 AI 是否能正确识别）**
3. **始终验证参数合法性，防止注入攻击**
4. **支持多轮函数调用（如先查日志再重启服务）**
5. **记录调用日志，便于调试和审计**

### 2、常见误区提醒：

- ❌ 认为“AI 能直接运行代码” → 实际只能建议调用
- ❌ 忽视参数校验 → 可能导致安全风险
- ❌ 把所有逻辑都塞进 prompt → 应该用函数拆分职责

## 结语

Function Calling 是连接大模型与现实世界的桥梁。它让我们能够构建真正有用的 AI 应用，而不仅仅是聊天机器人。掌握这项技术，你就拥有了打造智能运维、自动化客服、数据分析助手等高级系统的钥匙。
