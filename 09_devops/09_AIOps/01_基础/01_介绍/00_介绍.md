# 介绍

## 一、什么是AIOps

>定义：使用人工智能技术简化运维工作的技术方案，包括：机器学习、数据分析等技术。

![image-20251221230104229](https://picture-bed.gmbaifa.online/PicGo/image-20251221230104229.png)

## 二、优势

>- 持续观测监控数据流：AIOps对数据处理的吞吐量和敏感度远超人工
>- 解决运维团队孤岛问题：AIOps能够通过业务指标、日志、追踪数据关联性随时发现问题
>- 预测问题并提前解决：借助机器学习预测问题，而不是被动的解决问题
>- 根因分析：借助机器学习技术快速处理大数据，并确定问题的根本原因

## 三、基础：数据接入和运维接口能力

>- 运维接口能力：重启服务、回滚配置、调整参数、终止发布、回滚版本。依赖k8s的声明式更好实现。
>- 建模/分析：数据分析、数据建模、深度歇息、大模型分析
>- 数据存储：时序数据库、数据仓库、实时分析型数据库、关系型数据库
>- 数据接入：业务日志、网关日志、分布式追踪、业务级指标、系统级指标、拨测数据、健康检查数据、元数据
>- 业务应用：虚拟机应用、容器应用、无服务器应用、函数服务应用、边缘计算应用。

## 四、大模型和AIOps

>大模型的出现极大降低了数据建模和分析的难度
>
>文心一言、chatgpt、LLAMA、通义千问、Gemini

### 1、传统OPS的工作方式

>框时间轴，找到故障时刻所有的指标和日志的方式递归查下游分析原因
>
>- 人工分析，数据量大，专家级要求
>- 经验驱动

### 4、将大模型应用到AIOps的理论基础

![image-20251221232335231](https://picture-bed.gmbaifa.online/PicGo/image-20251221232335231.png)

>- Multi-Agent：多个Agent分工和相互协作
>- Agent：自主思考、决策、行动
>- Chains：模型链式调用，上一个输出作为下一个输入。
>- Prompt：提示词工程，例如zero-shot、few-shot
>- 模型层：各种大模型API

### 5、将大模型应用到AIOps的核心技术

>- Fine-tuning:微调
>- RAG：检索增强生成
>- Function Calling：函数调用（链式、意图识别等）
>- JSON Mode：大模型稳定输出JSON
>- Prompt Engineering：提示词质量决定输出质量

### 6、大模型在AIOps的技术挑战

>- 运维场景严肃性决定对错误容忍度低，大模型幻觉无法完全解决
>- 数据隐私、安全性、合规等限制，大模型需支持低成本的私有部署
>- 运维高质量的标注数据、语料和知识库不足
>- 很难处理大量的实时数据（上下文Token、成本限制）
>- 如何结合存量DevOps运维工具

### 7、LLM+AIOps可观测性

>- 集成LangSmith 实现LLM Tracing（SaaS）
>
>- 开源替代：Langfuse
>- 查看Input（Prompt）、Output、Tools调用和调用结果
>- 基于Tracing内容在线调试（Prompt+上下文）

>
>核心原理：接触词频（TF）-逆文本频率指数（IDF）得到一段日志的特征向量
