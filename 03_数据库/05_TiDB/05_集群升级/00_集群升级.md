# 集群升级

## 一、打补丁

```bash
tiup cluster patch xiaowu-cluster /tmp/XX.tar.gz -R tidb
tiup cluster patch xiaowu-cluster /tmp/XX.tar.gz -R tikv
tiup cluster patch xiaowu-cluster /tmp/XX.tar.gz -N NODE_IP:NODE_PORT
```

## 二、版本升级

### 1、升级TiUP

```bash
tiup update --self
tiup update cluster
```

### 2、修改TiUP CLuster拓扑配置文件

```bash
tiup clustger edit-config xiaowu-cluster
```

### 3、检查健康状态(regions)

```bash
tiup check xiaowu-cluster --cluster
```

### 4、集群升级到制定版本

#### 1.不停机

```bash
tiup cluster upgrade xiaowu-cluster v5.2.1
```

#### 2.停机

```bash
tiup cluster stop xiaowu-cluster
tiup cluster upgrade xiaowu-cluster v5.2.1 --offline
tiup cluster start xiaowu-cluster
```

### 5、检查状态

```bash
tiup cluster display xiaowu-cluster
```

## 三、升级常见问题

### 1、升级时报错终端如何继续升级

#### 1.查看操作记录，找到失败的升级操作记录的ID

```bash
tiup cluster audit
```

#### 2.重试上次的升级操作记录

```bash
tiup cluster replay AUDIT_ID
```

### 2、升级过程中evict leader等待时间过长，如何跳过该步骤升级

```bash
tiup cluster update oldguo-cluster v5.2.1 --force
```
