# 延时从库

## 一、介绍

```mysql
是我们认为配置的一种特殊从库.人为配置从库和主库延时N小时.
主库做了某项操作后，从库延时多长时间回放。可以处理逻辑损坏。
```



## 二、为什么要有延时从库

```
普通的主从复制，处理物理故障损坏比较擅长。
如果主库出现drop databases操作形成逻辑损坏，普通主从无法处理。

延时从库：主库做了某项操作后，从库延时多长时间回放。可以处理逻辑损坏。
```



## 三、配置延时从库

```mysql
SQL线程延时:数据已经写入relaylog中了,SQL线程"慢点"运行
一般企业建议3-6小时,具体看公司运维人员对于故障的反应时间

查看普通主从是否正常
mysql> show slave status \G

停止从库
mysql>stop slave;
mysql>CHANGE MASTER TO MASTER_DELAY = 300;
mysql>start slave;

查看配置是否成功：
mysql> show slave status \G
SQL线程回放延时时间，单位s（秒）
SQL_Delay: 300
SQL_Remaining_Delay: NULL 下次业务执行时间
```



## 四、故障模拟及恢复

### 1、模拟数据

```mysql
create database ys charset utf8mb4;
use ys;
create table t1(id int);
insert into t1 values(1),(2),(3),(4);
insert into t1 values(11),(22),(33),(44);
insert into t1 values(111),(222),(333),(444);

稍后执行
drop database ys;
```



### 2、恢复思路整理

```mysql
1.先停相关库业务，挂维护页。
2.停止从库sql线程
	stop slave sql_thread;
	等主库所有日志量同步完，看relay.info ---> 位置点
	stop slave;
	
3.追加后续缺失部分的日志到从库（手工模拟SQL工作）
	日志在哪：relay-log
	
	范围：relay.info ---> drop
	
4.恢复业务的方案
	1.ys导出恢复到主库
	2.推荐方法：直接将从库直接承当ys的业务
```

#### 2）主库删库

```mysql

```



#### 3）停止相关业务，挂维护页

#### 4）停止从库SQL线程

```mysql
stop slave sql_thread;
```



#### 5）等待一会，让残余ys业务日志复制到从库。

#### 6）停止从库所有线程（IO）

```mysql
stop slave IO_thread;
```



#### 7）手工恢复数据到drop之前

##### 1.截取relaylog

###### 1）找到截取点

```mysql
起点：
	方法一
	Relay_Log_File: db02-relay-bin.000002
    Relay_Log_Pos: 1439

	方法二
	[root@db02 ~]# cat /service/mysql/data/relay-log.info 
    ./db02-relay-bin.000002
    1439
    mysql-bin.000004
    1273

终点：
	mysql> show relaylog events in 'db02-relay-bin.000002';


     Log_name		log文件名
     Pos			relay-log 起点position号
     Event_type		时间类型
     Server_id		服务id
     End_log_pos	主库bin-log 起点position号
     Info  			发生的事件
     
     综上，终点：
				1504 |drop database ys
```



###### 2）截取日志

```mysql
[root@db02 ~]# mysqlbinlog --start-position=1439 --stop-position=1504 /service/mysql/data/db02-relay-bin.000002 >/tmp/relay.sql
```



###### 3）从库恢复

````mysql
set sql_log_bin=0;
source /tmp/relay.sql;
mysql> set sql_log_bin=1;

````
