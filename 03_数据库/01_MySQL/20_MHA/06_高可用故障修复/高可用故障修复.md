# 故障修复思路

**故障模拟：停掉主库**

```bash
[root@db01 ~]# systemctl stop mysqld.service
```



## 一、排查进程状态

```bash
[root@db03 ~]# ps -ef |grep [m]anager
[root@db03 ~]# masterha_check_status --conf=/etc/mha/app1.cnf
```



## 二、检查配置文件中节点情况

```bash
[root@db03 ~]# cat /etc/mha/app1.cnf
[server default]
report_script=/usr/local/bin/send_report
master_ip_failover_script=/usr/local/bin/master_ip_failover
manager_log=/var/log/mha/app1/manager
manager_workdir=/var/log/mha/app1
master_binlog_dir=/service/mysql/binlog
user=mha
password=mha
ping_interval=2
repl_password=123
repl_user=repl
ssh_user=root

[server2]
hostname=10.0.0.52
port=3306

[server3]
hostname=10.0.0.53
port=3306

[binlog1]
no_master=1
hostname=10.0.0.53
master_binlog_dir=/data/mysql/binlog
```

**如果节点信息消失，说明切换过程已经大部分成功。如果节点还在，证明切换过程在中间卡住**



## 三、看日志

```bash
tail -f /var/log/mha/app1/manager
```



## 四、修复

### 1、修复故障库

```bash
systemctl start mysqld.service
```



### 2、将故障库修好后，手工加入已有的主从中，作为从库

```bash
change master to
master_host='10.0.0.52',
master_user='repl',
master_password='123',
MASTER_AUTO_POSITION=1;
start slave;

show slave status\G
```

**如果故障库宕机时间过长，先备份恢复在做从**

### 3、修复MHA配置文件

```bash
[root@db03 ~]# vim /etc/mha/app1.cnf

[binlog1]
hostname=10.0.0.53
master_binlog_dir=/data/mysql/binlog
no_master=1

[server default]
manager_log=/var/log/mha/app1/manager
manager_workdir=/var/log/mha/app1
master_binlog_dir=/service/mysql/binlog
master_ip_failover_script=/usr/local/bin/master_ip_failover
password=mha
ping_interval=2
repl_password=123
repl_user=repl
report_script=/usr/local/bin/send_report
ssh_user=root
user=mha

[server1]
hostname=10.0.0.51
port=3306

[server2]
hostname=10.0.0.52
port=3306

[server3]
hostname=10.0.0.53
port=3306

```



### 4、检查ssh互信和repl的主从关系

```bash
[root@db03 ~]# masterha_check_ssh --conf=/etc/mha/app1.cnf
[root@db03 ~]# masterha_check_repl --conf=/etc/mha/app1.cnf
```



### 5、修复binlogserver

#### 1）删除无用日志

```bash
[root@db03 ~]# cd /data/mysql/binlog/
[root@db03 /data/mysql/binlog]# rm -rf *
```



#### 2）查看新主库当前日志文件

```bash
db02 [(none)]>show master status;
...
	mysql-bin.000008
...	
```



#### 3）拉取新主库的二进制文件

```bash
[root@db03 /data/mysql/binlog]# mysqlbinlog -R --host=10.0.0.52 --user=mha --password=mha --raw --stop-never mysql-bin.000008 &

```

**草稿**

```bash
mysqlbinlog -R --host=172.16.0.26 --user=mha --password=mha --raw --stop-never mysql-bin.000002 &
```



### 6、检查VIP

```bash
[root@db02 ~]# ip a
如果不在，手动生成一下
```



### 7、启动MHA

```bash
[root@db03 ~]# nohup masterha_manager --conf=/etc/mha/app1.cnf --remove_dead_master_conf --ignore_last_failover < /dev/null > /var/log/mha/app1/manager.log 2>&1 &
[2] 16288

[root@db03 ~]# masterha_check_status --conf=/etc/mha/app1.cnf
app1 (pid:16288) is running(0:PING_OK), master:10.0.0.52

```
