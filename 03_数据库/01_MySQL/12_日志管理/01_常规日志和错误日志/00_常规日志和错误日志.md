# 常规日志（general_log）

## 一、作用

>一般不会开启开功能，因为log的量会非常庞大。但个别情况下可能会临时的开一会儿general log以供调试使用。

## 二、查询及配置

```mysql
show variables like 'general_log'; -- 查看日志是否开启
show variables like 'general_log_file'; -- 看看日志文件保存位置
show variables like 'log_output'; -- 看看日志输出类型 table或file
set global general_log=on; -- 开启日志功能
set global general_log_file='/data/logs/general.log'; -- 设置日志文件保存位置
set global log_output='table'; -- 设置输出类型为 table
set global log_output='file'; -- 设置输出类型为 file
set global log_output='table,file'; -- 设置多位置保存
```

## 三、使用

>检测mysqldump备份过程。



# 错误日志（log_error）

## 一、作用

```mysql
1.记录MySQL从启动以来，所有的状态，警告，错误。
2.为我们定位数据库问题，提供帮助
```



## 二、文件位置

```mysql
1.默认：开启状态，在数据路径下
2.查看数据路径：
	mysql> select @@datadir;
    +----------------------+
    | @@datadir            |
    +----------------------+
    | /service/mysql/data/ |
    +----------------------+
3.文件名：
	主机名.err
```



## 三、自己定制

```mysql
定制方法：
	vim /etc/my.cnf
	log_error=
		绝对路径（名字也可以自定义，提前存在且mysql有写入权限,同目录下不能有重名文件）
	重启生效
	
如果报错尝试用mysqld --default

```



## 四、怎么使用

```mysql
1.数据库无法启动时，看错误日志
2.找中括号是error，查看上下文整行无法启动的原因
```
